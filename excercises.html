1. Introduction to Programming in Python

ðŸ“˜ Theory (Deeper Concepts)

An Algorithm is the logical blueprint (finite, unambiguous steps) for solving a problem, independent of any language. The Program is the source code implementation of that algorithm. Syntax is the language's grammar (rules); Semantics is the code's meaning (logic). Python uses an Interpreter (specifically a Python Virtual Machine that executes intermediate bytecode), which translates and executes code line-by-line at runtime, making development rapid but execution typically slower than compiled languages.

2. Tools for Program Development

ðŸ“˜ Theory (Deeper Concepts)

IDEs (like PyCharm) provide advanced features like context-aware code completion and integrated debugging, enhancing developer productivity far beyond a simple text editor. Git is a distributed version control system that tracks the history of every line of code; GitHub is the popular cloud platform for collaborating on and hosting Git repositories. Virtual environments isolate project dependencies, ensuring project A doesn't break when installing a new version needed by project B.

3. Data Types, Operators, and Terms

ðŸ“˜ Theory (Deeper Concepts)

Python's Numeric types handle numbers with special consideration for float precision due to binary representation issues. Sequence types (list, tuple) maintain order. Lists are mutable (can change in place), while Tuples are immutable, making them ideal for fixed data. Dictionaries (dict) use hashing on immutable keys to achieve $O(1)$ (constant time) average retrieval speed, making them exceptionally fast for lookups.

ðŸ’» Exercises (Deeper Focus)

  1. Demonstrate the floating-point precision issue by calculating 0.1 + 0.2 and explaining why the result isn't exactly 0.3.
  2. Calculate the volume of a cylinder given a radius $r$ and height $h$, using a combination of arithmetic operators and the float type.
  3. Use comparison and logical operators (and, or, not) to check if a user's age is between 18 and 65, inclusive, but not exactly 30.

4. Input and Output (I/O)

ðŸ“˜ Theory (Deeper Concepts)

The core concept of I/O is the interaction with external resources. The input() function always returns a string, necessitating type casting (e.g., int(input())) for numerical data. The print() function uses arguments like sep (separator) and end (ending character) to control output appearance beyond the default space and newline. f-strings are evaluated at runtime, embedding variable values as the string is constructed.

  2. Compute the sum and product of two numbers input by the user, displaying the results in a single print() statement using the sep argument.
  3. Read a monetary value from the user (as a string), convert it to a float, and use f-string formatting to display it rounded to two decimal places and prefixed with a dollar sign.

5. Flow Control

ðŸ“˜ Theory (Deeper Concepts)

Flow control determines the program's path through conditional execution (if/elif/else) or repetition (loops). The for loop is definite iteration (knows how many times to run, usually over a sequence). The while loop is indefinite iteration (runs until a condition is met). break terminates the entire loop; continue skips only the current iteration.

ðŸ’» Exercises (Deeper Focus)

  1. Write a program that finds the largest of three numbers using only nested if statements, then simplify it using and and or logical operators.
  2. Use a for loop with the range() function and an if statement to print all even numbers between 1 and 50, then rewrite it using a single while loop.
  3. Compute the factorial of a user-input number using a for loop, then rewrite it using a while loop, and compare their syntactic clarity.

6. Program Structures

ðŸ“˜ Theory (Deeper Concepts)

Modular programming involves breaking code into distinct, reusable Modules (.py files). Packages are directories of related modules, essential for structuring large projects and preventing name clashes. Namespaces are mapping systems (like dictionaries) that keep track of names (variables, functions) and the objects they reference, ensuring no ambiguity. Scope (Local, Enclosing, Global, Built-in - LEGB) dictates which namespaces are searched, and in what order, when resolving a variable name.

ðŸ’» Exercises (Deeper Focus)

  1. Create a module named utils.py with functions for calculating area and perimeter. In your main script, import utils and call one of the functions to demonstrate module usage.
  2. Create a folder hierarchy to structure a package with a main file and two sub-modules, and use dot notation (e.g., from package.submodule import func) to call a function.
  3. Write a function with a variable named x, and an outer script with a global variable x. Inside the function, print x and explain which scope's x was used based on the LEGB rule.

7. Functions

ðŸ“˜ Theory (Deeper Concepts)

Functions promote Don't Repeat Yourself (DRY) code by encapsulating logic for reuse. Default arguments (e.g., def func(a=1)) must be defined after non-default arguments. Recursion is a powerful technique for solving problems that can be broken into smaller, identical subproblems, but it requires a careful base case to prevent infinite loops (Stack Overflow). Lambda functions are restricted to a single expression that is returned implicitly, making them concise for simple operations.

ðŸ’» Exercises (Deeper Focus)

  1. Write a function that computes the average of a list, ensuring it handles the case of an empty list to prevent a ZeroDivisionError (using an if statement).
  2. Implement the recursive Fibonacci sequence function, tracing the function calls for fib(4) to clearly illustrate the call stack and the base case.
  3. Use filter() with a lambda function to select and return only the odd numbers from a given list of integers.

8. References and Pointers (Python Context)

ðŸ“˜ Theory (Deeper Concepts)

Python variables are names bound to objects. When you assign b = a, you create a second reference (label) to the exact same object in memory. Mutability (the ability of an object to change in place) is key: changing a mutable object affects all names referencing it, while reassigning an immutable object simply binds the name to a new object. The id() function provides the unique memory address (identity) of an object.

ðŸ’» Exercises (Deeper Focus)

  1. Initialize a list a = [1, 2] and set b = a. Modify b[0] = 99. Print both a and b to show the reference effect (both changed). Then, start over, use a = b.copy(), modify b, and show that a is now independent.
  2. Write a function that takes a list, appends an item to it, and does not return anything. Call the function using a global list and explain why the global list was permanently modified.
  3. Demonstrate why 1000 is 1000 might return False but 5 is 5 returns True (due to Python's optimization for small, frequently used integers).

9. Data Structures

ðŸ“˜ Theory (Deeper Concepts)

Choosing the right data structure is critical for performance. Lists are best for ordered sequences where insertion and deletion are common. Tuples are best for fixed records or when used as dictionary keys (because they are immutable and hashable). Sets are optimized for membership testing and removing duplicates quickly. Dictionaries are the primary mapping type, utilizing keys for fast, non-sequential access.

  1. Store and retrieve student grades in a dictionary where the key is the student ID (an int) and the value is a list of their scores. Write code to add a new score and then print the average for a specific student.
  2. Create two sets, one of required courses and one of completed courses. Find the set of courses the student still needs to take (set difference) and the set of all unique courses taken or needed (set union).
  3. Implement a Queue (First-In, First-Out) using a list, demonstrating the append() and pop(0) methods, and discuss why using a collections.deque is more efficient for this purpose.

10. Searching and Sorting

ðŸ“˜ Theory (Deeper Concepts)

Searching finds a target element; Sorting arranges elements in order. Time Complexity measures how the runtime grows with the input size $n$. Linear Search is $O(n)$, while Binary Search is $O(\log n)$, making it exponentially faster for large, sorted data. Simple sorts like Bubble Sort are $O(n^2)$ (quadratic), whereas professional algorithms like Merge Sort and Timsort (used by Python) achieve $O(n \log n)$, which is considered efficient.

ðŸ’» Exercises (Deeper Focus)

  1. Write a Linear Search function and run it on a list of 1 million numbers, timing the result to find a number at the beginning vs. the end to demonstrate $O(n)$.
  2. Implement the Bubble Sort algorithm. Modify your code to count the total number of swaps and comparisons needed to sort a list of 100 randomly generated numbers.
  3. Use the built-in list.sort() method on a list of 10,000 strings. Then, write a custom comparison function (key) to sort the same list by string length instead of alphabetical order.

11. Strings

ðŸ“˜ Theory (Deeper Concepts)

Python strings are immutable sequences of Unicode characters, allowing them to handle most global languages. Because they are immutable, operations like concatenation (s + "more") create entirely new string objects. Slicing (s[start:stop:step]) allows for precise extraction of substrings. Methods like .split() and .join() are often used together to parse complex text formats and then reconstruct them.

ðŸ’» Exercises (Deeper Focus)

  1. Write a program to count vowels in a string, ensuring it handles both upper and lower-case letters.
  2. Check if a string is a palindrome (reads the same forwards and backwards) by using negative slicing ([::-1]) for an efficient one-line solution.
  3. Given a URL string (e.g., "https://www.google.co.uk/search"), use a combination of .split(), .replace(), and indexing/slicing to extract and print only the root domain name (google).
